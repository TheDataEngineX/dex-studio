{# ── How watermarks work ── #}
How watermarks work

Watermark cursor — after each successful pipeline run the scheduler records the latest updated_at or ingested_at timestamp for that source. On the next run only records newer than this cursor are fetched, enabling incremental ingestion.

Content-hash deduplication — a SHA-256 hash of each ingested row is stored. If the same row appears again in a subsequent run it is silently dropped. The hash count column shows how many unique rows have been ingested from each source.

Reset — clears both the cursor and all stored hashes for a source. The next pipeline run will ingest the full history. Use with caution on large sources.
